## Reporting Bugs

### Before Submitting a Bug Report

Please make sure that:
- You are using the latest version of the app
- The issue is not caused by a local setup or configuration problem
- The bug has not already been reported in the [issue tracker](/issues?q=label%3Abug)

Helpful information to include:
- What you expected to happen
- What actually happened
- Steps to reproduce the issue
- Platform (iOS / Android / Web)
- Expo version and Node.js version
- Screenshots or screen recordings (especially for UI issues)

---

### How Do I Submit a Good Bug Report?

> âš ï¸ **Do not report security vulnerabilities publicly.**  
> Please report sensitive issues privately to the maintainers.

To submit a bug report:
- Open a new [Issue](/issues/new)
- Clearly describe the expected behavior and the actual behavior
- Include clear reproduction steps
- Attach logs, screenshots, or error messages when possible

Once submitted:
- A maintainer will review and label the issue
- If it can be reproduced, it may be marked `needs-fix`
- Issues marked `good first issue` are great entry points for new contributors

## Suggesting Enhancements

This section covers feature ideas, UI improvements, and general enhancements.

### Before Submitting an Enhancement

- Make sure you are using the latest version
- Check whether the feature has already been suggested
- Consider whether the enhancement fits the scope of Hack Club Mobile
- Think about whether it benefits most users

---

### How Do I Submit a Good Enhancement Suggestion?

Enhancement suggestions are tracked using [GitHub Issues](/issues).

Please:
- Use a clear and descriptive title
- Describe the current behavior
- Explain the improvement you would like to see
- Explain why this enhancement would be useful
- Include screenshots or mockups if applicable
